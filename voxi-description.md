# VOXI — Описание проекта и исследования

---

## Что такое Voxi

**Voxi** — AI-платформа для автоматизации продаж. Бизнес подключает свои источники лидов (WhatsApp, Instagram, сайт), все заявки попадают в единый CRM с Kanban-доской, а AI-боты автоматически звонят лидам, квалифицируют их и двигают карточки по воронке. Живые менеджеры подключаются только к тёплым лидам.

**Целевая аудитория:** казахстанские B2B-компании — агентства недвижимости, автосалоны, медклиники, образовательные центры. Компании с командой продаж от 2 до 50 менеджеров, где лиды приходят из разных каналов и теряются без системы.

**Бизнес-модель:** SaaS-подписка в тенге, три тарифа от 25 000 до 150 000 ₸/мес. Доп. доход — минуты звонков сверх лимита.

## Как работает продукт

### Общая схема

```
Источники лидов
──────────────────────────────────────────────
WhatsApp → ─┐
Instagram → ─┤──→ Voxi CRM (Kanban) ──→ AI Бот ──→ Звонок лиду
Сайт API  → ─┘         │                              │
                        │                              ↓
                   Менеджеры          Транскрипт + итог сохраняется
                   (живой чат)        Карточка двигается по воронке
```

### Полный путь лида

1. **Заявка поступает** — из WhatsApp, Instagram DM или формы на сайте. Автоматически создаётся карточка в колонке "Новый".

2. **Бот звонит** — в течение 30 секунд AI-бот инициирует исходящий звонок через казахстанский номер. Бот знает контекст: откуда пришёл лид, имя, что искал.

3. **Разговор** — бот разговаривает на русском языке, квалифицирует лида (бюджет, потребность, сроки), отвечает на вопросы из базы знаний компании. Поддерживает barge-in (когда клиент перебивает), делает паузы-заполнители ("Понял вас...", "Секунду...").

4. **Итог** — транскрипт звонка и AI-резюме сохраняются на карточке. Карточка автоматически перемещается в нужную колонку ("Квалифицирован", "Отказ", "В работе"). Менеджер получает уведомление.

5. **Менеджер подключается** — если лид тёплый, менеджер продолжает общение напрямую через WhatsApp прямо из интерфейса Voxi.

6. **Если не дозвонились** — бот повторяет попытку через заданное время (например, через 30 мин, потом через 2 часа), максимум N попыток.

### Настройка бота клиентом

Клиент через интерфейс задаёт:
- Имя и голос бота
- Промпт (инструкции: как представиться, что спрашивать, как реагировать на возражения)
- Базу знаний (PDF с прайсом, FAQ, описание услуг) — бот отвечает на вопросы из этих документов
- Расписание звонков (только в рабочие часы)
- Количество попыток дозвониться
- Что делать после звонка (в какую колонку переместить)

## Исследования

### Телефония для Казахстана

**Проблема:** Нужны казахстанские номера для исходящих звонков. Местные операторы (Beeline KZ, Tele2) подключают 2-4 недели и требуют бюрократии.

**Исследованные варианты:**

| Провайдер | Статус для KZ | Скорость | Стоимость номера |
|-----------|--------------|----------|-----------------|
| **Telnyx** | ✅ National KZ номера | Минуты через API | ~$1/мес |
| Twilio | ✅ KZ есть | Дни | Дороже ~30% |
| Beeline KZ | ✅ Местный (мобильный) | 2-4 недели | По договору |
| Tele2 KZ | ✅ Местный (мобильный) | 2-4 недели | По договору |

**Решение:** Telnyx на старте — быстро, есть API, поддерживает SIP Trunk для подключения к VAPI. Параллельно получать номер от Beeline/Tele2 для продакшна, т.к. мобильные номера имеют выше answer rate чем National тип.

**Риски Telnyx KZ:**
- Тип "National" — не мобильный (+7 700/701/705). Часть абонентов может не брать трубку с незнакомого формата номера
- Новый аккаунт может быть заблокирован антифрод системой — нужна верификация через support с описанием use case

### Voice AI — VAPI vs собственный пайплайн

**Из чего состоит голосовой звонок AI:**
```
Речь клиента
     ↓
STT (Speech-to-Text) — расшифровка речи в текст
     ↓
LLM (языковая модель) — генерирует ответ
     ↓
TTS (Text-to-Speech) — синтез речи из текста
     ↓
Аудио клиенту
```

**Задержка реального разговора:**
```
VAD детектит паузу:        200-500ms
STT транскрибирует:        200-400ms
LLM генерирует ответ:      500-1500ms
TTS синтезирует:           200-400ms
──────────────────────────────────────
Итого реальный мир:        1.0 — 2.5 сек
С оптимизированным стеком: ~700ms
```

**VAPI (vapi.ai) — готовый оркестратор:**
- Склеивает STT → LLM → TTS в единый pipeline
- Умеет: barge-in detection, backchanneling ("угу", "понял"), transfer на менеджера, webhooks в CRM, RAG из документов
- Поддерживает Telnyx как SIP провайдер
- Время интеграции: 1 день

**Стоимость одной минуты разговора через VAPI:**
| Компонент | Стоимость |
|-----------|-----------|
| VAPI оркестрация | $0.05/мин |
| STT (Deepgram) | ~$0.01/мин |
| LLM (GPT-4o mini) | ~$0.02–0.20/мин |
| TTS (ElevenLabs) | ~$0.04/мин |
| Telnyx телефония | ~$0.01/мин |
| **Итого** | **$0.13–0.31/мин** |

**Собственный пайплайн:**
- ~500–800 строк кода для базовой версии
- 3–4 недели до стабильного MVP
- Нужно самим решать: VAD, jitter buffer, session state, barge-in
- Имеет смысл при объёме >10 000 минут/месяц

**Решение:** VAPI на старте. После роста — переписать на Go + LiveKit (open-source media server, та же база что у Retell AI).

**Оптимальный стек по скорости:**
- STT: **Deepgram Nova-3** (режим Multi для русского)
- LLM: **GPT-4o mini** (быстрый первый токен) / Claude для сложных сценариев
- TTS: **Cartesia Sonic** (~90ms) или ElevenLabs (эмоции, ~400ms)

**Казахский язык:** Deepgram Nova-3 Multi теоретически поддерживает казахский, но качество заметно хуже русского. На MVP — только русский. Казахский добавить позже через Gladia.

**Как делают конкуренты:**
- **Retell AI** — Go media server + Python AI pipeline, ~600ms latency, BYOC Telnyx/Twilio
- **VAPI** — TypeScript SDK снаружи, Go+Rust внутри для voice infrastructure
- **LiveKit** — open source Go media server, основа многих платформ, включая ElevenLabs Conversational AI

### Платёжная система (B2B подписки в тенге)

**Контекст задачи:** Принимать ежемесячные платежи от казахстанских бизнесов в KZT за SaaS-подписку с автосписанием.

**Stripe и Paddle — не подходят:**
- Stripe не работает с казахстанскими юрлицами. KZT не поддерживается
- Paddle принимает продавцов из KZ, но KZT нет в ~30 поддерживаемых валютах. Комиссия 5% + $0.50 — невыгодно

**Казахстанские провайдеры:**

| Провайдер | Комиссия | Подписки | Node.js | Вывод |
|-----------|----------|----------|---------|-------|
| **ForteBank** | 2–2.5% | ✅ Plans + Subscriptions API | REST | 🥇 Лучший для SaaS |
| **TipTop Pay** | от 2.5% | ✅ Встроенный движок | npm пакет | 🥈 Альтернатива |
| **ioka** | 2.9% | ⚠️ Только токенизация | REST | Лучший конверт |
| **Halyk ePay** | 2.5–3% | ⚠️ Через cardSave | npm виджет | Большой охват |
| **Kaspi Pay** | 3% | ❌ Нет авторекуррента | Нет | Только доп. канал |

**ForteBank — почему лучший вариант:**
- Единственный банк с полноценным Subscription Plans API (создать план → подписать клиента → автосписание с retry)
- Документация на английском (docs.fortebank.com) — лучшая среди казахстанских банков
- Комиссия 2–2.5%, зачисление на следующий день включая выходные
- PCI DSS берёт на себя виджет, хранить данные карт не нужно

**ioka — почему интересен как дополнение:**
- Мультиэквайринг через 6 банков — автоматически выбирает банк с лучшим approve rate
- Клиенты: Chocofamily, Aviata, Досcar — проверенная нагрузка
- Нет Subscription API — всю логику подписок нужно писать самому (scheduler + retry + уведомления)

**Полноценного аналога Stripe Billing в Казахстане не существует.** ForteBank — ближайший вариант, но subscription management (dunning, proration, аналитика MRR) всё равно нужно строить самому.

### Технический стек — итоговое решение

**Фаза 1 (MVP):** TypeScript монолит на Next.js
- Проще деплоить, одна кодовая база, TypeScript везде
- Next.js App Router — frontend + API routes в одном
- MongoDB — гибкая схема, хорошо для CRM с кастомными полями
- BullMQ + Redis — очереди для звонков и биллинга

**Фаза 2 (после первых клиентов):** Микросервисы

| Сервис | Язык | Причина |
|--------|------|---------|
| Voice Media Server | Go | goroutines, настоящий multi-thread, низкий latency — так делают Retell/LiveKit |
| STT + LLM + TTS | Python | Deepgram/ElevenLabs/LangChain — нативные AI библиотеки |
| CRM + Kanban + Биллинг | TypeScript | Next.js, основной стек |
| Интеграции (WhatsApp/IG) | TypeScript | Все SDK нативно JS |

**Почему Go для voice, а не Node.js:**
- Node.js однопоточный — при 100+ параллельных звонках появляется риск блокировки
- Python GIL убивает параллелизм в CPU задачах
- Go: goroutine per call, легко масштабируется
