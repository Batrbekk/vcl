## 1. Краткий анализ проекта

**VOXI** — это AI-платформа для автоматизации продаж в Казахстане. Суть продукта:

- Бизнес подключает источники лидов (WhatsApp, Instagram, сайт)
- Все заявки попадают в единую CRM с Kanban-доской
- AI-бот автоматически звонит лидам через казахстанский номер, квалифицирует их, сохраняет транскрипт
- Карточка лида автоматически перемещается по воронке
- Живые менеджеры подключаются только к тёплым лидам

**Целевая аудитория:** казахстанские B2B-компании — агентства недвижимости, автосалоны, медклиники, образовательные центры. Команды продаж от 2 до 50 менеджеров.

**Бизнес-модель:** SaaS-подписка в тенге (25 000 — 150 000 KZT/мес), доп. доход на минутах сверх лимита.

**Техническое решение (по документам):**
- MVP = Next.js монолит (App Router) + MongoDB + Redis/BullMQ
- Голосовой движок = VAPI.ai (оркестратор STT->LLM->TTS) + Telnyx (SIP/телефония KZ)
- Платежи = ForteBank (единственный с Subscription API в KZ)
- Деплой = Vercel (app) + Railway (workers)

**Мой вывод:** Документация проработана глубоко — архитектура, модели данных, API-маршруты, даже latency-оптимизация Voice AI. Это документация на 12-недельный MVP. Но у нас есть 1 день. Нужно жёстко определить scope демо.

## 2. Критические вопросы для основателя

### А. Демо — что именно показываем?

1. **Кому показываем демо завтра?** Инвесторы, потенциальные клиенты, партнёры? От этого зависит, что делать — кликабельный прототип или рабочую систему.

2. **Какие экраны ОБЯЗАТЕЛЬНО должны быть в демо?** Из полного списка (dashboard, kanban, звонки, настройки бота, база знаний, интеграции, биллинг, настройки, команда) — что показываем, а что нет?

3. **Демо должно быть с реальными звонками или достаточно показать записанный/симулированный звонок?** Реальный звонок через VAPI+Telnyx = нужны аккаунты, настройка, верификация. Это риск за 1 день.

4. **Достаточно ли показать pre-recorded видео/аудио звонка + интерфейс с fake data?** Или нужен live-demo формат "вот прямо сейчас бот звонит"?

5. **Есть ли конкретный сценарий демо?** Например: "Приходит заявка из WhatsApp -> появляется на Kanban -> бот звонит -> транскрипт появляется -> карточка перемещается". Если да — опиши по шагам.

### Б. Аккаунты и API

6. **Какие аккаунты уже зарегистрированы?**
   - VAPI.ai — есть аккаунт? API ключ?
   - Telnyx — есть аккаунт? Казахстанский номер получен?
   - OpenAI — есть API ключ?
   - ElevenLabs / Cartesia — есть аккаунт для TTS?
   - MongoDB Atlas — есть кластер?
   - Vercel — есть аккаунт?
   - Meta (WhatsApp/Instagram Business API) — есть доступ?
   - ForteBank — есть мерчант-аккаунт?

7. **Есть ли домен?** (voxi.kz / app.voxi.kz?) Есть ли SSL? Если нет — демо будет на localhost или на Vercel preview URL?

8. **Есть ли сервер/VPS или всё будет на облачных сервисах?** Для демо — хватит ли Vercel free tier?

### В. Scope демо — что must-have, что nice-to-have

9. **Kanban-доска с лидами** — must-have для демо? Нужен ли drag-and-drop или достаточно визуального отображения колонок?

10. **Журнал звонков с транскриптом** — показываем? Если да — достаточно ли fake-данных (предзаполненные записи с текстовым транскриптом)?

11. **Аналитика/дашборд** — нужен ли на демо? (графики: количество звонков, конверсия, средняя длительность)

12. **Настройки AI-бота** — показываем UI для настройки промпта/голоса? Или это можно пропустить?

13. **WhatsApp-чат в интерфейсе** — показываем? Или это отдельный спринт?

14. **Биллинг/тарифы** — показываем страницу с планами? Или это чисто визуал, без ForteBank?

15. **Авторизация** — нужен ли экран логина или демо открытое (без пароля)?

### Г. Данные для демо

16. **Для какой индустрии делаем демо-сценарий?** Агентство недвижимости? Автосалон? Медклиника? Нужно знать, чтобы сгенерировать правильные fake-данные (имена лидов, запросы, транскрипты).

17. **Сколько fake-лидов нужно в Kanban?** 5-10 достаточно или нужно 50+ для "ощущения реального продукта"?

18. **Нужен ли демо-аккаунт конкретной компании?** (например, "АН Нурболат" из документации)

### Д. Дизайн

19. **Есть ли макеты в Figma / дизайн-система?** Или делаем на shadcn/ui с минимальным кастомом?

20. **Есть ли лого, брендбук, цвета?** (Вижу logo.svg и logo-text.svg в репозитории — это финальные?)

21. **Десктоп only или мобильная версия тоже нужна для демо?**

## 3. Рекомендации CTO

### Стек для демо (сегодня-завтра)

| Компонент | Решение | Причина |
|-----------|---------|---------|
| **Framework** | Next.js 15 (App Router) | Как в архитектуре, SSR + API в одном |
| **UI** | shadcn/ui + Tailwind | Быстрая сборка, красивые компоненты из коробки |
| **Kanban DnD** | @dnd-kit/core или статичные колонки | Drag-and-drop делать только если есть время |
| **База данных** | Пока без БД — хардкод JSON / in-memory | Для демо не нужна MongoDB, это overhead |
| **Auth** | Пропустить или mock-login (один пароль) | Тратить время на Auth.js — нет смысла для демо |
| **Звонки** | Fake data + аудиоплеер с записанным разговором | Реальные звонки = высокий риск сбоя на демо |
| **Деплой** | Vercel (free tier) | Моментальный деплой, preview URL |

### Стратегия: "Impressive Fake"

За 1 день невозможно построить рабочий продукт из документации на 12 спринтов. Моя рекомендация:

1. **Делаем визуально законченный фронтенд** — dashboard, kanban, журнал звонков, карточка лида с транскриптом
2. **Все данные — захардкожены** — 10-15 лидов, 5-7 звонков с транскриптами, аналитика с графиками
3. **Записываем 1-2 демо-звонка заранее** (если есть VAPI аккаунт) и встраиваем аудио в интерфейс
4. **Добавляем анимации** — появление нового лида, перемещение карточки, "живой" плеер записи звонка

Это создаст впечатление работающего продукта без риска "а что если API упал в момент демо".

### Если основатель настаивает на live-звонке

Тогда минимум нужно:
- Аккаунт VAPI с API ключом (регистрация 5 минут)
- Аккаунт Telnyx с казахстанским номером (может занять часы на верификацию)
- Или использовать **VAPI Web Call** (звонок через браузер, без телефонии) — это проще и надёжнее для демо

## 5. Риски

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Не успеваем всё за день | Высокая | Приоритет: kanban > звонки > dashboard. Если не успеваем dashboard — показываем kanban + звонки |
| Основатель хочет live-звонки | Средняя | VAPI Web Call через браузер — fallback без телефонии |
| Нет дизайна/макетов | Высокая | shadcn/ui даёт достаточно качественный UI из коробки |
| Vercel деплой падает на демо | Низкая | Держать localhost как backup |
